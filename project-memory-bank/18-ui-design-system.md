# 18 — UI Design System (Domain)

> Authoritative for frontend/UX. Consult before any UI work. Apply the `/frontend-design` skill.

## Aesthetic Reference
UI must feel like **Linear, Stripe, GitHub, Datadog, Vercel** — engineering-grade developer tooling.

## Design Principles
- Premium · Professional · Trustworthy · Fast · Minimal · Accessible · Engineering-focused.

## Avoid
- Flashy UI, excessive animations, marketing aesthetics, consumer-app styling.

## Developer Console Surfaces
The SCP Developer Console should eventually contain:
- **System Overview** — health, status, key metrics.
- **Memory Explorer** — browse/inspect memories + trust metadata.
- **Knowledge Graph Explorer** — entities & relationships, traversal.
- **Trust Inspector** — trust scores, provenance, verification status.
- **Semantic Query Debugger** — inspect query plans, retrieval, ranking.
- **Agent Runtime Monitor** — live agent/context/tool activity.
- **Replay Viewer** — Flight Recorder replay & root-cause.
- **Governance Dashboard** — policies, compliance, controls.
- **Audit Logs** — full auditable history.

## Core UX Mandate
Everything must be **observable and explainable**. The console exists to make SCP's cognition transparent — every memory, fact, score, and decision should be inspectable down to its evidence.

## Accessibility
WCAG-conscious: keyboard navigation, sufficient contrast, semantic structure. Accessibility is a requirement, not a polish step.

> No frontend stack is selected yet; this file governs *how* the UI should look/behave when built, not *with what*. Stack choice will be recorded via ADR when UI work begins.
